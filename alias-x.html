Project Overview

An AI-assisted academic credential verification system designed to help identify fake or invalid certificates through OCR extraction, human review, institutional verification and secure reporting.

Problem Addressed

Manual certificate verification can be slow, difficult to trace and dependent on repeated communication with institutions. ALIAS_X combines extraction, verification and reporting in one workflow.

My Contribution

Designed the workflow, integrated OCR and AI-assisted verification modules, implemented editable fields and built the report-generation and dashboard flow.

PROJECT WORKFLOW

How the system works

1

User Login

Verifier or administrator signs in.

2

Upload Certificate

The academic certificate is uploaded.

3

OCR Extraction

Name, register number, college, course, grades and dates are extracted.

4

Manual Review

The user checks and corrects extracted fields.

5

Verification Request

The system contacts the institution or registrar.

6

Phone Verification

An automated call requests confirmation.

7

Email Fallback

Email verification is used when the call does not confirm.

8

Response Analysis

The response is interpreted and classified.

9

Final Decision

The result is marked Verified, Rejected or Pending.

10

PDF Report

A downloadable verification report is generated.

11

Store Results

Certificate data and verification records are saved.

12

Dashboard

Status badges and previous records are displayed.

Key Features

  • OCR-based certificate field extraction
  • Manual review and correction
  • AI-assisted phone verification
  • Email verification fallback
  • Verified, Rejected and Pending status tracking
  • Secure PDF report generation
  • Dashboard and verification history
